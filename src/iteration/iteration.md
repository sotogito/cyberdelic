# Iteration Module
### 점 하나를 수식으로 반복하여 갱신

`iteration`은 같은 규칙을 여러 번 적용해서 문양을 "성장"시키는 과정이다.
Cyberdelic에서는 반복이 단순 복사가 아니라, 각 스텝에서 미세하게 변형되는 누적 연산이다.

## 1) 왜 반복이 필요한가

- 프랙탈/트랩 패턴의 복잡도는 한 번 계산으로 나오지 않는다.
- 반복할수록 중심, 가지, 경계가 분리되며 문양의 깊이가 생긴다.
- 반복 횟수는 곧 도안의 밀도와 긴장도를 결정한다.

## 2) 반복의 기본 구조

도안 생성 루프의 핵심은 아래 순서다.

1. 현재 상태 `z`를 준비한다.
2. 변환 규칙 `f(z, params)`를 적용한다.
3. trap 거리/조건을 평가한다.
4. 탈출(escape) 또는 수렴(converge) 여부를 검사한다.
5. 다음 스텝으로 상태를 넘긴다.

즉, `z(n+1) = f(z(n))` 를 조건이 끝날 때까지 반복한다.

## 3) 주요 파라미터

- `maxIter`: 최대 반복 횟수 (복잡도 상한)
- `escapeRadius`: 탈출 판정 반지름 (보통 2.0 이상)
- `seed`: 동일 결과 재현을 위한 난수 시드
- `variation`: 반복 중 변형 강도 (회전, 스케일, 왜곡)
- `trapWeight`: trap 조건이 색/선 굵기에 반영되는 비율

## 4) 반복 횟수와 시각 결과

- 반복 횟수 낮음: 단순, 굵고, 로고형 실루엣
- 반복 횟수 중간: 리듬과 결이 살아나는 균형형
- 반복 횟수 높음: 섬세하지만 과밀 위험 증가

타투 실사용 기준에서는 "읽히는 복잡도"가 중요하므로,
무조건 반복을 늘리기보다 출력 크기에 맞는 `maxIter`를 선택한다.

## 5) trap과 iteration의 관계

- iteration은 형태를 확장하고,
- trap은 각 반복 단계에서 "어디가 긴장 포인트인지"를 측정한다.

실무적으로는 반복 루프 안에서 trap 값을 갱신해 최솟값/최댓값을 추적한다.
이 누적값이 최종 명암, 선 굵기, 강조 영역을 만든다.

## 6) 구현 메모 (p5.js)

- 픽셀 루프마다 `z`를 초기화하고 iteration을 실행한다.
- 계산 단계와 렌더링 단계를 분리한다.
  - 계산: `iterCount`, `trapMin`, `escaped` 저장
  - 렌더: 저장된 값을 색상/두께로 매핑
- 성능이 부족하면:
  - 해상도 다운샘플
  - `maxIter` 동적 조절
  - 고정 구간 캐싱

## 7) 최소 의사코드

```js
z = z² + c
```
1. 처음 z0
2. 다음 z1 = z0² + c
3. 다음 z2 = z1² + c
4. 다음 z3 = z2² + c

---

iteration은 "반복 횟수"가 아니라 "형태가 자라는 시간축"이다.
Cyberdelic에서는 iteration을 통해 리듬, 밀도, 긴장을 제어한다.
