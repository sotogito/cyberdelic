# Trap Module

`trap`은 도안의 시선을 붙잡고 흐름을 전환하는 핵심 구조다.
사이버 트라이벌 문맥에서 trap은 단순 장식이 아니라, 리듬을 끊고 다시 연결하는 신호 결절점(node)으로 동작한다.

## 1) 역할

- **리듬 고정:** 반복 패턴 중 특정 지점을 강조해 전체 문양의 박자를 만든다.
- **시선 유도:** 선의 진행 방향을 꺾거나 수렴시켜 시선을 중심부로 끌어들인다.
- **밀도 제어:** 고밀도 영역과 여백 영역의 경계를 정의해 도안이 숨 쉬게 한다.
- **구조 연결:** `fractal`, `track`, `mirror` 모듈 사이의 전환 포인트로 작동한다.

## 2) 형태 원칙

- 기본 실루엣은 갈고리(hook), 송곳(spike), 고리(loop), 절단된 아크(arc) 조합으로 구성한다.
- 바깥쪽은 공격적인 각도, 안쪽은 제어된 곡률로 대비를 만든다.
- trap 하나의 형태는 간결하게 유지하고, 군집 배치로 복잡도를 만든다.
- 미세 디테일보다는 중간 굵기의 선 대비를 우선해 타투 전사 안정성을 확보한다.

## 3) 생성 규칙 (Generative Rules)

- 입력 파라미터 예시:
  - `anchor` (기준점)
  - `direction` (진행 벡터)
  - `length` (주요 길이)
  - `bend` (굽힘 강도)
  - `teeth` (반복 돌기 수)
  - `symmetry` (대칭 여부/강도)
  - `noiseAmount` (변이 강도)
- 생성 순서:
  1. 기준 축(axis) 생성
  2. trap 골격(메인 스트로크) 생성
  3. 치아(teeth) 또는 보조 가시 추가
  4. 끝점 taper 처리
  5. 인접 모듈과 접합부 스무딩

## 4) 조합 방식

- `track -> trap -> track` 구조로 리듬 전환점을 만든다.
- `fractal`의 분기 끝점마다 소형 trap을 배치해 긴장감을 증가시킨다.
- `mirror` 적용 시 완전 대칭보다 90~95% 유사 대칭을 사용해 기계적 생동감을 유지한다.
- 레이어 구성 시 trap은 보통 mid-layer에 배치하고, core silhouette은 침범하지 않는다.

## 5) 타투 도안 품질 체크

- 1m 거리에서 trap 방향성이 읽히는가
- 축소(30~40%) 시 선 뭉침이 발생하지 않는가
- 스텐실 전사 시 끊어지는 급격한 미세 곡률이 없는가
- 피부 곡면(팔/종아리/등) 적용 시 흐름이 자연스러운가

## 6) 구현 메모 (p5.js)

- trap 생성 함수는 가능하면 geometry 데이터를 반환한다. (즉시 draw보다 재조합 우선)
- 예: `createTrap(params) -> { spine, teeth, outline }`
- 렌더링 단계에서만 `strokeWeight`, `blendMode`, `alpha`를 적용해 스타일과 구조를 분리한다.
- seed 기반 난수로 동일 파라미터 재생성을 보장한다.

---

trap은 문양의 공격성과 질서를 동시에 담당하는 장치다.
Cyberdelic에서는 trap을 "시선과 리듬을 포획하는 모듈"로 정의한다.
