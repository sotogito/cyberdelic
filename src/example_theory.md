# Orbit Trap 코드가 작동하는 방식

**한 줄 요약:**  
그림 = `모든 픽셀 순회` + `픽셀마다 반복식 계산` + `trap 거리로 색칠`

## 1) 전체 흐름

이 코드는 캔버스의 픽셀을 하나씩 검사한다.  
픽셀 하나마다 아래 과정을 수행한다.

1. 픽셀 위치 `(i, j)`를 복소수 좌표 `c`로 바꾼다.
2. `z = 0`에서 시작한다.
3. `z_{n+1} = z_n^2 + c`를 여러 번 반복한다.
4. 반복 중 `z`가 trap에 얼마나 가까운지 계산한다.
5. 가장 가까웠던 거리 `score`를 저장한다.
6. `score`를 밝기로 바꿔 현재 픽셀에 색을 칠한다.

즉, **픽셀 하나당 색 하나**가 결정된다.

## 2) 중요한 오해 정리

### 오해 A: `z = z^2 + c`가 화면을 직접 그린다?

아니다. 화면 전체를 훑는 것은 이중 반복문이다.

```js
for (let i = 0; i < img.width; i++) {
  for (let j = 0; j < img.height; j++) {
    // 픽셀 하나 처리
  }
}
```

`z = z^2 + c`는 **픽셀 하나의 색을 결정하기 위한 내부 계산**이다.

### 오해 B: 내부 반복 30번이면 점 30개를 찍는다?

아니다.

```js
for (let index = 0; index < 30; index++) {
  // z를 30번 갱신하며 score 계산
}
```

이 30번은 화면에 점 30개를 찍는 게 아니라,  
현재 픽셀의 `score`를 계산하기 위해 `z`를 30번 움직여보는 것이다.  
실제로 색칠되는 것은 마지막의 **현재 픽셀 하나**다.

## 3) 이 코드의 trap은 무엇인가?

trap 핵심 식은 다음이다.

```js
abs(1 - z.norm())
```

`z.norm()`은 `|z|` (원점에서의 거리)다.  
즉 trap 식은 사실상 아래와 같다.

```txt
abs(1 - |z|)
```

뜻: **반지름 1짜리 원에 얼마나 가까운가**.  
그래서 이 코드는 점 trap이 아니라 **원형 trap**을 사용한다.

## 4) score는 어떻게 정해지나?

처음 점수는 `q0 = 1`로 시작한다.

```js
let s = q0;
```

반복할 때마다 trap 거리(`t`)를 계산하고, 더 작은 값만 저장한다.

```js
s = min(s, abs(1 - z.norm()));
```

즉 `s`의 의미는:

> 이 픽셀의 반복 궤도 `z`가 반지름 1 원에 가장 가까웠던 거리

- 가까이 지나가면 `s`가 작다.
- 멀리 지나가면 `s`가 크다.

## 5) 색은 어떻게 칠하나?

계산된 `s`로 픽셀 밝기를 정한다.

```js
if (c < q0) {
  img.set(i, j, color(255 - map(c, 0, 0.06, 0, 255)));
}
```

흐름:

1. 반복 궤도가 trap에 가까이 지나감 -> 작은 `score`
2. 작은 `score`를 밝기로 변환
3. 현재 픽셀 `(i, j)`에 색칠

중요: 색칠하는 위치는 `z`의 위치가 아니라 **원래 픽셀 위치 `(i, j)`**다.

## 6) 왜 프랙탈 같은 무늬가 생기나?

trap이 무늬를 직접 만드는 것은 아니다.  
큰 구조는 반복식에서 생긴다.

```txt
z_{n+1} = z_n^2 + c
```

각 픽셀마다 `c`가 다르므로, `z` 궤도도 달라진다.  
trap은 그 궤도들이 반지름 1 원에 얼마나 가까웠는지를 측정해 강조한다.

역할 분리:

- **반복식**: 전체 구조와 프랙탈 패턴 생성
- **trap**: 어떤 부분을 강조할지 결정
- **색 매핑**: 밝기, 대비, 질감 결정

## 7) 최종 한 줄 정리

이 코드는 모든 픽셀마다 `z_{n+1} = z_n^2 + c`를 반복하고,  
그 궤도가 반지름 1 원에 얼마나 가까웠는지를 점수로 저장한 뒤,  
그 점수로 픽셀 색을 정한다.


---
- 하나의 좌표에 점이 찍히는 과정
1. 현재 좌표에서 (0,0) 반복식을 사용하여 z값을 30번 구함
2. 그 값중 trap에 가장 가까운 값을 뽑음 = score
3. score을 기준으로 점을 찍음

아래는 하나의 좌표값이 이동되는 반복식의 좌표값
``` js
function setup() {
  createCanvas(400, 400);
  background(0);
  noLoop();

  // 보고 싶은 좌표 (여기 값을 바꾸면 궤적이 바뀜)
  let cx = -0.4;
  let cy = 0.6;

  drawOrbit(cx, cy);
}

function drawOrbit(cx, cy) {
  let zx = 0;
  let zy = 0;

  for (let i = 0; i < 50; i++) {

    // z = z^2 + c
    let newX = zx*zx - zy*zy + cx;
    let newY = 2*zx*zy + cy;

    zx = newX;
    zy = newY;

    // 복소수 좌표 → 화면 좌표
    let px = map(zx, -2, 2, 0, width);
    let py = map(zy, -2, 2, 0, height);

    // 궤적 점 찍기
    stroke(255);
    point(px, py);

    // 연결선 (흐름 보기)
    if (i > 0) {
      line(prevX, prevY, px, py);
    }

    prevX = px;
    prevY = py;

    // 너무 멀어지면 중단
    if (sqrt(zx*zx + zy*zy) > 2) break;
  }
}
```
