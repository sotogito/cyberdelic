# Orbit Trap

크게는 아래의 요소로 그림이 결정된다.
1. 반복식
2. trap
3. score값의 매핑(색, 크기)

---

## 1. 반복식

전체 구조를 만드는 핵심 식입니다.

```txt
z_{n+1} = z_n^2 + c
```

이 식이 `z`의 궤적(orbit)을 만듭니다.

그림 결과는 보통 아래 요소들의 조합으로 결정됩니다.

- 반복식
- 좌표 매핑
- 초기값
- 반복 횟수
- 발산 조건
- trap 모양
- score 계산 방식
- 색 매핑
- 픽셀 출력 방식

## 2. 좌표 매핑

화면 픽셀을 계산용 좌표 `c`로 바꾸는 단계입니다.

```js
cx = map(x, 0, width, -2, 2);
cy = map(y, 0, height, -2, 2);
```

범위를 바꾸면 확대, 축소, 이동 효과가 납니다.

## 3. 초기값

반복을 어디서 시작할지 정합니다.

```txt
z = 0
```

만델브로 방식은 보통 `z = 0`에서 시작합니다.

## 4. 반복 횟수

한 픽셀당 반복식을 몇 번 계산할지 정합니다.

```js
for (let i = 0; i < 30; i++)
```

반복 횟수가 많을수록 더 복잡하고 세밀한 구조가 나옵니다.

## 5. 발산 조건

너무 멀리 나간 궤적은 계산을 멈춥니다.

```js
if (|z| > 2) break;
```

이 조건은 계산량을 줄이고, 바깥 영역을 구분하는 역할을 합니다.

## 6. trap 모양

`z`가 어떤 도형에 가까운지 재는 기준입니다.

예시:

```txt
abs(1 - |z|)
```

이 식은 반지름 1인 원에 얼마나 가까운지를 재는 원형 trap입니다.

다른 trap 예시:

- 점까지 거리
- 선까지 거리
- 원까지 거리
- 여러 도형 거리의 조합

## 7. score 계산 방식

반복 중 나온 거리들 중 어떤 값을 최종 점수로 쓸지 정합니다.

가장 흔한 방식:

```js
score = min(score, trapDistance);
```

의미:

- 반복 궤도가 trap에 가장 가까웠던 순간의 거리

다른 방식도 가능합니다:

- 마지막 거리
- 평균 거리
- 누적 거리
- 처음 가까워진 순간
- 여러 trap 거리 조합

## 8. 색 매핑

`score`를 실제 픽셀 색으로 바꾸는 단계입니다.

```js
brightness = map(score, 0, 0.06, 255, 0);
```

여기서 그림의 분위기, 대비, 선명도, 색감이 결정됩니다.

## 9. 픽셀 출력 방식

계산 결과를 어디에 찍을지 정합니다.

이 코드에서는:

```js
img.set(x, y, color);
```

즉, 궤적 위치가 아니라 **원래 픽셀 위치**에 색을 칠합니다.
